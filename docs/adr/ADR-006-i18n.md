# ADR-006: Internacionalización (i18n) con next-intl, locale por cookie

**Fecha:** 2026-06-10
**Estado:** Aceptada (fundación / parcial)

## Contexto

Todos los textos estaban hardcodeados en español. Para habilitar la expansión
(en especial **portugués/Brasil**, ver `docs/lista-de-deseos.md` y Fase 3 del
`CLAUDE.md`) hay que internacionalizar. Esta ADR sienta las **bases**; la
conversión del resto de las pantallas es incremental.

## Decisión

- **Librería:** `next-intl` (estándar para el App Router de Next 16).
- **Estrategia de locale: por COOKIE (`NEXT_LOCALE`), sin prefijo en la URL.**
  Se eligió por encima del prefijo `/[locale]/...` para **no romper los links
  existentes** `/c/[slug]` (que el negocio ya pudo imprimir/compartir) ni tener que
  mover toda la app a `app/[locale]/`.
- **Idiomas:** `es` (default), `en`, `pt`.

### Piezas
- `src/i18n/config.ts` — locales, default, nombres, helper `isLocale`.
- `src/i18n/request.ts` — `getRequestConfig` que lee la cookie y carga el catálogo.
- `next.config.ts` — envuelto con `createNextIntlPlugin('./src/i18n/request.ts')`.
- `src/app/layout.tsx` — `<html lang={locale}>` + `<NextIntlClientProvider>`.
- `messages/{es,en,pt}.json` — catálogos.
- `src/components/features/locale-switcher.tsx` — selector que setea la cookie y
  refresca.

### Alcance convertido en esta fundación
- **Flujo del cliente final**: landing `/c/[slug]` + `phone-form` (alto valor; es
  lo que ve el consumidor). El resto del dashboard sigue en español por ahora.

## Pendiente (incremental)
1. Convertir el resto de pantallas (dashboard, onboarding, etc.). El **flujo del
   cliente final ya está traducido**: landing, formulario OTP y **tarjeta**.
2. **Errores de las server actions**: hoy devuelven texto en español. Migrar a
   **códigos** y traducir en la UI (o pasar `locale`). Mientras tanto, los errores
   de servidor se muestran en español aunque el resto esté traducido.
3. Detección automática por `Accept-Language` en la primera visita (hoy: default
   `es` hasta que el usuario elija).
4. Revisar el aviso legal (Habeas Data) por país: el texto `es` cita la Ley 1581
   de Colombia; `en`/`pt` usan un texto genérico.

## Consecuencias
- Agregar un idioma nuevo = un `messages/<locale>.json` + sumarlo a `locales`.
- Convertir una pantalla = reemplazar literales por `t('clave')` (server:
  `getTranslations`, client: `useTranslations`) y agregar las claves a los 3 JSON.
- Relacionado con multi-país (moneda/timezone/código de país por tenant) — ver la
  lista de deseos.
